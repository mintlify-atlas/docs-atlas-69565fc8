The recipe generation API provides the core abstraction for converting project models into intermediate conda recipes. By implementing `GenerateRecipe`, you can create custom build backends for different project types.

## GenerateRecipe Trait

Defined in: `src/generated_recipe.rs:36`

The `GenerateRecipe` trait is responsible for converting a `ProjectModel` into an `IntermediateRecipe`. Implementations of this trait define how specific project types (Python, CMake, etc.) are built.

```rust
#[async_trait::async_trait]
pub trait GenerateRecipe {
    type Config: BackendConfig;
    
    async fn generate_recipe(
        &self,
        model: &ProjectModel,
        config: &Self::Config,
        manifest_path: PathBuf,
        host_platform: Platform,
        python_params: Option<PythonParams>,
        variants: &HashSet<NormalizedKey>,
        channels: Vec<ChannelUrl>,
        cache_dir: Option<PathBuf>,
    ) -> miette::Result<GeneratedRecipe>;
    
    fn extract_input_globs_from_build(
        &self,
        config: &Self::Config,
        workdir: impl AsRef<Path>,
        editable: bool,
    ) -> miette::Result<BTreeSet<String>>;
    
    fn default_variants(
        &self,
        host_platform: Platform,
    ) -> miette::Result<BTreeMap<NormalizedKey, Vec<Variable>>>;
}
```

```rust
    NoNameDefined,
    NoVersionDefined,
    MetadataProviderError(String, MetadataProviderError),
}
```

Errors that can occur during recipe generation:

- `NoNameDefined` - Package name is missing from both model and provider
- `NoVersionDefined` - Package version is missing from both model and provider  
- `MetadataProviderError` - The metadata provider failed to extract a field

## Usage Example

Implementing a custom backend:

```rust
use pixi_build_backend::{
    GenerateRecipe, GeneratedRecipe, BackendConfig, PythonParams,
};
use pixi_build_types::ProjectModel;
use serde::Deserialize;

#[derive(Debug, Clone, Deserialize)]
struct MyBackendConfig {
    build_flags: Vec<String>,
}

impl BackendConfig for MyBackendConfig {
    fn debug_dir(&self) -> Option<&Path> {
        None
    }
    
    fn merge_with_target_config(&self, target: &Self) -> miette::Result<Self> {
        Ok(Self {
            build_flags: target.build_flags.clone(),
        })
    }
}

struct MyBackend;

#[async_trait::async_trait]
impl GenerateRecipe for MyBackend {
    type Config = MyBackendConfig;
    
    async fn generate_recipe(
        &self,
        model: &ProjectModel,
        config: &Self::Config,
        manifest_path: PathBuf,
        host_platform: Platform,
        python_params: Option<PythonParams>,
        variants: &HashSet<NormalizedKey>,
        channels: Vec<ChannelUrl>,
        cache_dir: Option<PathBuf>,
    ) -> miette::Result<GeneratedRecipe> {
        // 1. Extract metadata from the project model
        // 2. Convert dependencies to recipe requirements
        // 3. Generate build scripts based on config
        // 4. Create IntermediateRecipe
        // 5. Determine input globs
        
        Ok(GeneratedRecipe {
            recipe: intermediate_recipe,
            metadata_input_globs: metadata_globs,
            build_input_globs: build_globs,
        })
    }
    
    fn extract_input_globs_from_build(
        &self,
        _config: &Self::Config,
        _workdir: impl AsRef<Path>,
        _editable: bool,
    ) -> miette::Result<BTreeSet<String>> {
        Ok(BTreeSet::from([
            "src/**/*.cpp".to_string(),
            "CMakeLists.txt".to_string(),
        ]))
    }
}
```
