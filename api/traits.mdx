## PackageSpec Trait

Defined in: `src/traits/package_spec.rs:29`

The `PackageSpec` trait defines the interface for package specifications, which describe how to obtain and match packages.

```rust
pub trait PackageSpec: Send {
    type SourceSpec: PackageSourceSpec;
    
    fn can_be_used_as_variant(&self) -> bool;
    fn to_match_spec(
        &self,
        name: PackageName,
    ) -> miette::Result<(MatchSpec, Option<Self::SourceSpec>)>;
}
```

### Associated Types

<ParamField path="SourceSpec" type="PackageSourceSpec">
  The source representation of a package (e.g., path spec, git spec)
</ParamField>

### Methods

#### `can_be_used_as_variant()`

```rust
fn can_be_used_as_variant(&self) -> bool
```

Returns `true` if the package spec can be used as a variant specification. A spec can be used as a variant if it's a binary spec with:
- Version set to `*` (any version)
- No other constraints (build, build_number, channel, etc.)

<ResponseField name="return" type="bool">
  Whether this spec is suitable for use in variant configurations
</ResponseField>

---

#### `to_match_spec()`

```rust
fn to_match_spec(
    &self,
    name: PackageName,
) -> miette::Result<(MatchSpec, Option<Self::SourceSpec>)>
```

Converts the package spec to a conda `MatchSpec` for dependency resolution.

<ParamField path="name" type="PackageName">
  The package name to include in the match spec
</ParamField>

<ResponseField name="return" type="(MatchSpec, Option<SourceSpec>)">
  A tuple containing:
  1. The conda `MatchSpec` for matching packages
  2. An optional `SourceSpec` if the package should be built from source
</ResponseField>

## Targets Trait

Defined in: `src/traits/targets.rs:75`

The `Targets` trait represents project target configurations, allowing platform-specific dependencies and settings.

```rust
pub trait Targets {
    type Selector: TargetSelector;
    type Target;
    type Spec: PackageSpec;
    
    fn default_target(&self) -> Option<&Self::Target>;
    fn empty_spec() -> Self::Spec;
    fn targets(&self) -> impl Iterator<Item = (&Self::Selector, &Self::Target)>;
    fn dependencies(&self, platform: Option<Platform>) -> Dependencies<'_, Self::Spec>;
    fn run_dependencies(&self, platform: Option<Platform>) -> IndexMap<&SourcePackageName, &Self::Spec>;
    fn host_dependencies(&self, platform: Option<Platform>) -> IndexMap<&SourcePackageName, &Self::Spec>;
    fn build_dependencies(&self, platform: Option<Platform>) -> IndexMap<&SourcePackageName, &Self::Spec>;
    fn resolve(&self, platform: Option<Platform>) -> impl Iterator<Item = &Self::Target>;
}
```

### Associated Types

<ParamField path="Selector" type="TargetSelector">

</ParamField>

<ParamField path="Target" type="any">

</ParamField>

<ParamField path="Spec" type="PackageSpec">

</ParamField>


### Methods

#### `matches()`

Determines if a target selector matches a given platform.

<ParamField path="platform" type="Platform">
  The platform to test against
</ParamField>

<ResponseField name="return" type="bool">
  `true` if the selector matches the platform
</ResponseField>

<Expandable title="Selector Types">
The implementation supports several selector types:
- `Platform("linux-64")` - Exact platform match
- `Linux` - Matches any Linux platform
- `Unix` - Matches Linux and macOS
- `Win` - Matches Windows platforms
- `MacOs` - Matches macOS platforms
</Expandable>

## Usage Examples

### Checking for Variant-Compatible Specs

```rust
use pixi_build_backend::PackageSpec;

let spec = pbt::PackageSpec::Binary(VersionSpec::Any.into());
if spec.can_be_used_as_variant() {
    // This spec can be used in variant configurations
}
```

### Resolving Platform Dependencies

```rust
use rattler_conda_types::Platform;

let deps = targets.dependencies(Some(Platform::Linux64));
for (name, spec) in deps.build {
    println!("Build dependency: {}", name);
}
```

### Matching Target Selectors

```rust
use pixi_build_types::TargetSelector;

let selector = TargetSelector::Unix;
if selector.matches(Platform::Linux64) {
    // This target applies to Linux
}
```
