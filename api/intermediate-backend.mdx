The `IntermediateBackend` is a generic implementation of the protocol traits that works with any recipe generator. It handles the RPC protocol by converting project models to conda recipes and managing the build process.

## Architecture

The intermediate backend consists of two main components:

1. **IntermediateBackendInstantiator** - Implements `ProtocolInstantiator` for setup
2. **IntermediateBackend** - Implements `Protocol` for handling build requests

## Protocol Implementation

### `conda_outputs()`

Defined in: `src/intermediate_backend.rs:224`

Implements output discovery by:

1. Merging target-specific configuration for the host platform
2. Constructing variant configuration from defaults, files, and user input
3. Generating the intermediate recipe using `GenerateRecipe::generate_recipe()`
4. Converting the recipe to YAML source code
5. Expanding all variant combinations using `rattler-build`
6. Filtering skipped outputs
7. Converting each output to the protocol format
8. Writing debug artifacts (recipe.yaml, variants.yaml) to work directory

<Expandable title="Debug Output Structure">
The backend writes debug information to `{work_directory}/debug/`:
```
debug/
  recipe.yaml              # Generated intermediate recipe
  variants.yaml            # Full variant configuration
  recipe/
    {hash}/
      recipe.yaml          # Recipe for this specific variant
      variants.yaml        # Variant values for this output
```
</Expandable>

**Key Steps:**

```rust
// 1. Generate recipe
let generated_recipe = self.generate_recipe.generate_recipe(
    &self.project_model,
    &config,
    self.source_dir.clone(),
    host_platform,
    Some(PythonParams { editable: false }),
    &variants.keys().cloned().collect(),
    channels,
    self.cache_dir.clone(),
).await?;

// 2. Discover variants
let discovered_outputs = variant_config.find_variants(
    &outputs,
    named_source,
    &selector_config,
)?;

// 3. Convert to protocol format
for discovered_output in discovered_outputs {
    let recipe = Recipe::from_node(&discovered_output.node, selector_config)?;
    outputs.push(CondaOutput { /* ... */ });
}
```

### `conda_build_v1()`

Defined in: `src/intermediate_backend.rs:545`

Implements the build process by:

1. Determining build and host platforms from prefixes
2. Merging target-specific configuration
3. Constructing variant configuration from build parameters
4. Generating the recipe with editable flag consideration
5. Finding the matching output using `find_matching_output()`
6. Setting up build directories with `conda_build_v1_directories()`
7. Writing debug artifacts
8. Configuring `rattler-build` tool settings
9. Running the build with `run_build()`
10. Extracting input globs from the build and recipe

<Expandable title="Build Directory Structure">
The backend creates a structured work directory:
```
{work_directory}/
  work/        # Source code and build artifacts
  host/        # Host environment (if not provided)
  build/       # Build environment (if not provided)
  cache/       # Cache directory (if not provided)
  output/      # Built packages (if not specified)
  debug/       # Debug artifacts
```
</Expandable>

**Configuration Options:**

```rust
let tool_config = Configuration::builder()
    .with_opt_cache_dir(self.cache_dir.clone())
    .with_logging_output_handler(self.logging_output_handler.clone())
    .with_testing(false)
    .with_keep_build(true)  // Pixi is incremental
    .with_environments_externally_managed(true)
    .with_allow_absolute_license_paths(true)
    .finish();
```

## Usage Example

Creating a custom backend:

```rust
use pixi_build_backend::{
    IntermediateBackendInstantiator,
    GenerateRecipe,
    BackendConfig,
};
use std::sync::Arc;

// Define your recipe generator
struct MyRecipeGenerator;

#[async_trait::async_trait]
impl GenerateRecipe for MyRecipeGenerator {
    type Config = MyBackendConfig;
    
    async fn generate_recipe(/* ... */) -> miette::Result<GeneratedRecipe> {
        // Your recipe generation logic
    }
}

// Create the instantiator
let generator = Arc::new(MyRecipeGenerator);
let logging_handler = LoggingOutputHandler::default();
let instantiator = IntermediateBackendInstantiator::new(
    logging_handler,
    generator,
);

// Use with the protocol server
// The instantiator handles initialize() calls from the frontend
```
