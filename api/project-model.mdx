The `ProjectModel` trait is the central interface for working with project models in Pixi Build Backends. Any backend that deals with projects (like those from pixi frontend) should implement this trait.

## Trait Definition

Defined in: `src/traits/project.rs:17`

```rust
pub trait ProjectModel {
    type Targets: Targets;
    
    fn targets(&self) -> Option<&Self::Targets>;
    fn dependencies(&self, platform: Option<Platform>) -> Dependencies<'_, Self::Targets::Spec>;
    fn used_variants(&self, platform: Option<Platform>) -> HashSet<NormalizedKey>;
    fn name(&self) -> Option<&String>;
    fn version(&self) -> &Option<Version>;
}
```

## Usage

The `ProjectModel` trait is typically used by backends to:

1. **Access project metadata** - Get the project name and version for recipe generation
2. **Query dependencies** - Retrieve platform-specific dependencies for builds
3. **Determine variants** - Identify which variant configurations are needed
4. **Navigate targets** - Access target-specific configurations
